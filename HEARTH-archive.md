- **v311:** **Fix Sports section "Maximum call stack size exceeded" crash (Cathal-reported bug).** Root cause: `fetchSportsForTeam` synchronously invoked its callback (`renderSports`) whenever a fetch for a team was already in flight; `renderSports` unconditionally calls `refreshSports(false)` at its end, which — since the team's cache still wasn't populated (the original fetch hadn't resolved) — called `fetchSportsForTeam` again, hitting the same synchronous-callback branch again, and so on, an unbounded synchronous recursion with no async boundary to break it. Triggered by re-rendering/reopening the Sports tab while a team's fetch was still pending (e.g. slower network, quick tab switch away and back). Fix: the already-fetching branch now just returns without calling back — the in-flight fetch's own `step()` already calls `cb()` once when it resolves, so a concurrent caller doesn't need (and must not trigger) another render. One-line change in `fetchSportsForTeam`, no data-model or UI change.
